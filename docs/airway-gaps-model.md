# Airway Gaps Engine Model

Status: approved for implementation  
Date: 2026-07-14  
Branch: `feature/engine-airway-gaps`

## Purpose

This model closes the anesthesia engine gaps that prevented Carson-Newman RSI and Standard IV induction rubrics from scoring mask ventilation, cricoid pressure, failed intubation attempts, and the consequence of preoxygenation.

The truth boundary is unchanged: callers set clinical actions and drivers; the engine derives minute ventilation, EtCO2, oxygen stores, PaO2, and SpO2. No new airway API writes a derived vital.

## Existing state preserved

- `effectiveNmbBlockade` remains the only neuromuscular-blockade state.
- Respiratory muscle capability remains derived from the existing TOF ratio generated by that state.
- Forced apnea, drug depression, complication drive, NMB-mediated muscle capability, and airway device remain independently queryable.
- Airway device remains the validated `mask`, `intubated`, or `extubated` axis.
- The fixed timestep remains 0.02 seconds with float32 stores.
- The existing oxygen-reservoir model remains authoritative.

The Round 1 audit demonstrated that the existing oxygen model is adequate: a room-air-prepared 70 kg patient under forced apnea crossed SpO2 95% at 27 seconds and 90% at 42 seconds; after three minutes at FiO2 1.0, the corresponding crossings were 460 and 473 seconds. This round does not replace that model.

## Airway procedure subsystem

`AirwayProcedureSystem` owns timed airway actions and their immutable histories. It is wired to the existing patient and ventilator and is ticked in this order:

```text
DrugSystem
AirwayProcedureSystem
VentilatorSystem
PatientPhysiology
AlarmSystem
ScenarioManager
```

It consumes no random numbers. An unused procedure system does not change physiology or shift the established RNG sequence.

The rig helpers expose it additively as `a`:

```js
buildRig(seed)     -> { p, d, v, a, s, core }
buildPhysRig(seed) -> { p, d, v, a, core }
```

Existing destructuring callers that omit `a` remain valid.

## Timed mask positive-pressure ventilation

### Public methods

```js
deliverMaskVentilation({
  durationSeconds = 30,
  tidalVolumeMl = 500,
  respiratoryRate = 12,
  cricoidPressure = false,
} = {})

stopMaskVentilation()
```

`deliverMaskVentilation()` is one discrete clinical action that schedules a fixed-duration episode. It is not a writable `ppvOn` boolean. The method returns the accepted episode ID, fixed-step start time, requested duration, mask airway state, settings, delivered minute ventilation, and convenience cricoid metadata.

The action is accepted only when:

- duration, tidal volume, and respiratory rate are finite and greater than zero;
- airway device is `mask`;
- no PPV episode is already active; and
- no intubation attempt is active.

A rejected call returns `{ ok: false, reason }` and creates no successful action entry.

The episode ends when its duration elapses or when explicitly stopped, reset, interrupted by laryngoscopy, or invalidated by an airway change. History preserves both planned and actual delivered duration.

### Ventilation derivation

```text
ppvMechanicalMinuteVentilation = float32(
  tidalVolumeMl * respiratoryRate / 1000
)

effectiveMinuteVentilation = max(
  ppvMechanicalMinuteVentilation,
  spontaneousMinuteVentilation
)
```

During PPV, the ventilator exposes the episode tidal volume, respiratory rate, measured minute ventilation, and mechanical minute ventilation. The existing gas-exchange derivations consume effective minute ventilation.

PPV does not set or clear forced apnea, alter drug depression, alter the NMB/TOF state, change the airway device, or write oxygen/CO2 vitals. Adequate PPV therefore substitutes for absent spontaneous effort through the same mechanical-support path used by controlled ventilation.

### PPV action records

The procedure event stream contains:

```text
mask_ppv_started
mask_ppv_completed
```

Start metadata contains episode ID, timestamp, airway state, planned duration, TV, RR, derived mechanical MV, and the convenience cricoid flag. Completion metadata contains fixed-step end time, actual delivered duration, and completion reason.

Snapshot history records contain:

```js
{
  episodeId,
  startTimeSec,
  endTimeSec,
  plannedDurationSec,
  deliveredDurationSec,
  airwayDevice,
  tidalVolumeMl,
  respiratoryRate,
  minuteVentilation,
  cricoidPressure,
  completionReason,
}
```

## Cricoid pressure

### Public methods

```js
applyCricoidPressure()
releaseCricoidPressure()
```

These are independent, discrete, fixed-step maneuvers. They are not dependent on PPV and do not require an active mask-ventilation episode.

The event stream contains:

```text
cricoid_pressure_applied
cricoid_pressure_released
```

Applying an already-active maneuver or releasing an inactive maneuver is idempotent and does not create duplicate scoreable events. Snapshot exposes `cricoidPressureActive` and copied `cricoidPressureHistory` records with apply/release timestamps and duration.

The `cricoidPressure` flag on a PPV episode remains convenience metadata. The discrete `cricoid_pressure_applied` record is the scoreable proof for RSI items 9 and 29. A correct RSI can therefore apply cricoid pressure and omit mask ventilation before the first laryngoscopy attempt.

**Explicit simplification — recorded maneuver only:** cricoid pressure has no physiologic effect in this round. It does not change airway resistance, aspiration probability, PPV delivery, laryngoscopy difficulty, or intubation outcome.

## Deterministic intubation attempts

### Scenario configuration

```js
airwayPlan: {
  failedIntubationAttempts: [1],
  intubationAttemptDurationSeconds: 30,
}
```

Attempt numbers are positive integers. Normalization removes invalid and duplicate entries and sorts the remaining list. Duration must be finite and positive.

Outcome is deterministic membership in `failedIntubationAttempts`. No intubation success roll consumes RNG. The same scenario definition, seed, and action timing therefore produce the same attempt and physiologic results.

### Public method

```js
attemptIntubation()
```

Starting an attempt requires a `mask` airway and no current attempt. The accepted call increments the count immediately and returns the attempt number, fixed-step start time, duration, and starting airway without revealing outcome before completion.

At attempt start:

1. Active mask PPV ends with `interrupted_by_intubation`.
2. `proceduralApneaActive` becomes true.
3. `proceduralApneaContribution` becomes 0.
4. All mechanical support is inhibited.
5. The airway remains `mask`.

At completion:

- Failure clears procedural apnea and support inhibition and leaves the airway `mask`.
- Success clears procedural apnea and support inhibition and invokes the validated `mask -> intubated` transition.
- Neither result starts a ventilator mode or changes ventilator settings.

The attempt event stream contains:

```text
intubation_attempt_started
intubation_attempt_failed
intubation_attempt_succeeded
```

Each record contains:

```js
{
  attemptNumber,
  startTimeSec,
  endTimeSec,
  durationSec,
  outcome,
  airwayBefore,
  airwayAfter,
  spo2Nadir,
  desaturatedBelow90,
  timeToSpo2_90Sec,
}
```

`timeToSpo2_90Sec` is elapsed time from attempt start to the first fixed-step sample below 90% SpO2. It is `null` if no such crossing occurs. This makes the physiologic consequence of preoxygenation legible in debrief data.

### Procedural apnea composition

Intubation apnea is separate from explicit forced apnea:

```text
centralDrive = forcedApneaContribution
             * drugDepressionContribution
             * complicationDriveContribution
             * proceduralApneaContribution

effectiveSpontaneousVentilationFraction = centralDrive
                                         * respiratoryMuscleCapability
```

This leaves all pre-existing sources independently queryable. The procedure source is 0 only during laryngoscopy. The single NMB state still supplies the only respiratory-muscle impairment.

Support inhibition is an independent procedure condition. During the attempt, both spontaneous ventilation and mechanical support are absent, so existing oxygen stores drain and existing CO2 physiology evolves naturally.

## ETO2

Snapshot exposes end-tidal oxygen as a derived percentage:

```text
eto2 = float32(clamp01(alveolarO2Fraction) * 100)
```

**Explicit simplification — alveolar proxy:** the engine has no separate sampled end-tidal oxygen compartment. ETO2 is a teaching approximation derived from the existing alveolar oxygen fraction and has no feedback into physiology.

## Snapshot contract

The live snapshot adds:

| Key | Type | Definition |
|---|---|---|
| `eto2` | number | Derived end-tidal oxygen percent |
| `mechanicalMV` | number | Current mechanical ventilation contribution |
| `effectiveMV` | number | Ventilation consumed by gas exchange |
| `proceduralApnea` | boolean | Intubation procedure currently suppresses spontaneous ventilation |
| `proceduralApneaContribution` | number | 0 during an attempt, otherwise 1 |
| `cricoidPressureActive` | boolean | Discrete cricoid maneuver is active |
| `cricoidPressureHistory` | array | Copied apply/release records |
| `ppvActive` | boolean | Timed mask PPV is active |
| `ppvEpisodeCount` | number | Accepted PPV episode count |
| `ppvCurrent` | object or null | Copied current episode |
| `ppvHistory` | array | Copied current/completed PPV records |
| `intubationInProgress` | boolean | Timed laryngoscopy is active |
| `intubationAttemptCount` | number | Attempts started |
| `lastIntubationOutcome` | string | `none`, `failed`, or `succeeded` |
| `intubationAttempts` | array | Copied attempt records including desaturation timing |

Objects returned by `snapshot()` are copies and cannot mutate engine state.

## Scoreable paths

`setAirwayDevice()` is an administrative/setup method. It is useful for fixtures, starting states, and instructor setup, but it is not clinical performance evidence.

Only `attemptIntubation()` produces `intubation_attempt_*` log entries. Round 3 scoring must require `intubation_attempt_succeeded`; reaching `intubated` through `setAirwayDevice()` scores zero for all intubation-attempt rubric items.

Ordering discriminators use fixed-step action timestamps:

- RSI item 11: no `mask_ppv_started` before the first `intubation_attempt_started`.
- Standard IV items 7/8: successful mask PPV before the rocuronium drug entry.
- RSI rescue: a failed attempt followed by cricoid application and mask PPV before the next attempt.
- Correct RSI cricoid: `cricoid_pressure_applied` exists before first attempt while no PPV start exists.

## Live instructor-console behavior

Before this change, **INTUBATE** immediately changed the device to `intubated` and automatically selected VCV. After this change, **INTUBATE** starts the timed `attemptIntubation()` path:

- the console displays an in-progress attempt and remaining procedure time;
- the airway remains `mask` during laryngoscopy;
- spontaneous and mechanical ventilation are absent during the attempt;
- the attempt can fail according to scenario configuration;
- the airway changes only after a successful completion; and
- ventilator mode must be selected/configured separately.

The full induction-to-emergence smoke case advances through attempt duration, verifies success, and explicitly enables VCV before continuing.

## Reset rules

Engine initialization, engine reset, live-runner rebuild, and scenario reset clear:

- active/current PPV and its history;
- cricoid active state and history;
- current/completed attempts and counter;
- procedural apnea and support inhibition; and
- procedure timestamps/listeners owned by the run.

The procedure reset does not create or clear an independent NMB variable. Existing drug/patient reset remains the only NMB reset path.

## Explicit simplifications and exclusions

1. **Delivered-MV PPV:** bag-mask ventilation is modeled as prescribed TV and RR, not individual squeezes.
2. **Adequate mask seal assumed:** leak and difficult ventilation are out of scope.
3. **No pressure injury/gastric insufflation:** PPV complications are not modeled.
4. **Max composition:** simultaneous mechanical and spontaneous ventilation use `max`, not breath-by-breath summation.
5. **Recorded-only cricoid:** cricoid pressure has no physiologic effect.
6. **Configured deterministic failure:** attempts fail by scenario list, not anatomy, skill, probability, or RNG.
7. **Unsupported laryngoscopy:** all ventilation is inhibited for the entire configured attempt duration.
8. **No CICO state machine:** failed attempts with desaturation cost are modeled; a distinct CICO pathway is not.
9. **Approximate ETO2:** alveolar oxygen fraction is used as an end-tidal proxy.
10. **No automatic post-intubation ventilation:** securing a device and starting mechanical support remain orthogonal actions.

## Required evidence

The implementation must keep all prior evidence green and add:

1. PPV support versus withheld PPV in equivalent apneic/paralyzed patients.
2. PPV fixed-step timestamps and delivered duration.
3. RSI pre-laryngoscopy PPV discriminator.
4. Standard IV PPV-before-rocuronium discriminator.
5. Failed first attempt with time cost, desaturation, unsecured airway, and successful second attempt.
6. Preoxygenated versus non-preoxygenated apnea curves and ETO2 greater than 90% after adequate preparation.
7. Bit-identical combined runs with midpoint samples during PPV and laryngoscopy.
8. Prior frozen parity and regression evidence.
9. Ordered RSI rescue chain: failed attempt, cricoid, PPV, second attempt success, with oxygenation recovery during PPV.
10. Correct RSI cricoid record with no PPV before first attempt.
11. Attempt desaturation timing: finite below-90 crossing without preoxygenation and no crossing in the preoxygenated equivalent.
12. Administrative `setAirwayDevice()` creates no attempt log and cannot be scored as intubation.
